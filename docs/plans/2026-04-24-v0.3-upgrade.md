# LexCapital v0.3 Upgrade Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Upgrade LexCapital from a runnable MVP/Phase-2 prototype into a publishable v0.3 benchmark release with calibrated scenarios, stronger audit gates, reproducible leaderboards, and clearer model-eval workflows.

**Architecture:** Keep the current deterministic replay + explicit prompt allowlist foundation. v0.3 should add scenario calibration assets and reporting around the existing core instead of rewriting the engine. The highest leverage work is sidecars, metadata completeness, leaderboard/run manifests, and CI-style verification.

**Tech Stack:** Python 3.11+, Pydantic v2, Typer CLI, pytest, YAML scenario packs, JSONL action sidecars.

## Release target

v0.3 should mean:

1. Scenario quality is measurable, not just valid.
2. Every public scenario has oracle and red-path sidecars.
3. Scenario metadata has `expected_skill`, `trap_type`, and `baseline_expectations` coverage.
4. Leaderboard outputs include enough run metadata to reproduce results.
5. README has one clean path for authors, model runners, and benchmark publishers.
6. Package/docs versioning is coherent.

## My recommended order

Do **not** start with more scenario writing. That feels productive but weakens the benchmark if calibration is missing.

Best order:

1. Version/docs alignment
2. Metadata audit
3. Metadata migration
4. Oracle sidecars
5. Red sidecars
6. Baselines
7. Run manifests
8. Publish gate
9. README/docs polish

This makes v0.3 defensible instead of just “v0.2 with more YAML.”
